# Security policy

Use synthetic or properly authorized data only. Do not commit payment-card data,
credentials, personal information, or production model artifacts.

Report vulnerabilities privately through GitHub Security Advisories. Do not open a
public issue containing secrets, personal data, or exploit details.

Before production use, add authentication and role-based authorization, TLS, managed
secrets, payload limits, audit logging, encrypted storage, dependency scanning,
backup/restore procedures, and an incident-response plan.
